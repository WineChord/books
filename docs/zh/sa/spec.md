# 后缀数组入门规格说明

这本书是一个用单道力扣式数组题学习后缀数组的单页交互教材。它不是算法
词条，也不是静态代码注释；目标是让完全零基础的读者能从“什么是后缀”一路
走到理解给定 C++ 解法为什么正确。

## 路由契约

默认路由是 `/books/sa/`，并默认展示中文内容。英文路由是
`/books/sa/en.html`。语言切换是页面紧凑标题行里的普通链接。

配套规格页位于 `/books/sa/spec.html`，中文规格页位于
`/books/zh/sa/spec.html`。

## 视觉契约

页面沿用 LeetCode 书的视觉系统：相同的紧凑系统字体、浅色书页背景、
terracotta 主按钮、兼容暗色模式的边框、小字号表格，以及首屏不展示的大
封面。页面不展示全站顶栏。

首屏必须是工作台，而不是宣传页。桌面首屏应能同时看到输入区、阶段切换、
核心讲解卡和至少一部分可视化表格。移动端可以纵向堆叠，但文本、按钮、
表格和 token 不得互相遮挡。

## 教学契约

中文页面面向完全初学者。任何核心术语第一次出现时都要解释：

1. 后缀：从某个位置开始一直走到数组结尾的整段。
2. 子数组：连续拿出的一小段。
3. 子数组和后缀的关系：从位置 `i` 开始的子数组，一定是后缀 `a[i..]` 的
   一个前缀。
4. rank：当前轮排序后给每个后缀贴的编号；编号相同表示当前已看的长度内
   还分不出来。
5. LCP：两个相邻后缀从开头起连续相同的长度。

页面要把大问题拆成五步：概念、坐标压缩、倍增排序、LCP、答案扫描。每步
必须回答三个问题：

1. 读者正在看什么；
2. 这一步为什么合法；
3. 它对应给定 C++ 代码里的哪个函数或变量。

## 交互契约

页面必须提供一个联动工作台，而不只是渲染最终结果：

1. 输入 1 到 10 个整数后，页面实时重建压缩数组、后缀数组、LCP 和答案。
   更长输入会被截断到 10 个值，保证表格可读。
2. 样例按钮至少覆盖尾部唯一、全相同、交替重复三类情况。
3. 点击任意起点或任意后缀行时，所有面板都同步高亮同一个起点。
4. “概念”阶段必须展示当前起点的后缀，以及这个后缀能产生哪些前缀式子数组。
5. “坐标压缩”阶段必须展示原值、压缩值和去重排序字母表，并解释压缩不改变
   相等关系和大小关系。
6. “倍增排序”阶段必须有轮次滑块；当前轮要展示 `(rank[i], rank[i+k])`，
   并解释二元组相同为什么会拿到同一个新 rank。
7. “LCP”阶段必须展示当前后缀和左右邻居的公共前缀长度，直观看到哪些 token
   是共同前缀。
8. “答案”阶段必须展示 `left`、`right`、`same`、`need`、是否越界，以及
   `same + 1` 对应的候选子数组。
9. 页面必须有一个小型自测区。切换阶段后，自测题和反馈随阶段变化；点击选项
   后给出明确解释，而不是只显示对错。
10. 当前阶段要同步到代码讲解：至少指出当前阶段对应 `compress`、`build1`、
    `build2` 或 `smallestUniqueSubarray`。

## 算法契约

页面把给定的 C++ 代码拆成四段解释。

`compress` 复制输入，排序、去重，再把每个原始值映射成从 1 开始的 rank。
它保留相等关系和大小关系，只是把值域变成紧凑字母表。

`build1` 使用标准倍增思想。在第 `k` 轮，每个后缀按
`(rank[i], rank[i+k])` 排序；如果第二段越过数组末尾，就用 `-1`。排序后，
二元组相同的后缀得到同一个新 rank。`k` 不断翻倍，直到所有后缀 rank 都
不同，或 `k >= n`。

`build2` 用 Kasai 扫描构造 LCP 数组。`lcp[p]` 表示 `sa[p]` 和 `sa[p-1]`
这两个相邻后缀的最长公共前缀。复用高度 `h` 让这一段保持线性复杂度。

`smallestUniqueSubarray` 使用后缀数组的邻居结论。任何从 `i` 开始的子数组
都是后缀 `a[i..]` 的一个前缀。在字典序里，和 `a[i..]` 共享最长前缀的后缀
一定是它的前一个或后一个后缀。因此：

```text
same = max(和前一个后缀的 LCP, 和后一个后缀的 LCP)
need = same + 1
```

如果 `need` 没有越过当前后缀边界，它就是这个起点的最短唯一子数组长度。
最终答案是所有起点里最小的 `need`。

发布前必须验证：

1. `npm run build` 通过。
2. `npm run check:content` 通过。
3. `npm run check:dist` 通过。
4. 浏览器打开 `/books/sa/` 无控制台错误。
