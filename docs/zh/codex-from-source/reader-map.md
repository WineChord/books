# 阅读地图

这一页把本书的目标从“带你读源码”改成“尽量提供等价于读源码后的理解”。你仍然可以打开固定到
commit 的源码链接，但主路径应当不依赖读者同时打开仓库。

## 两条阅读路径

| 路径 | 你需要做什么 | 读完后应当知道什么 |
| --- | --- | --- |
| 不打开源码阅读 | 阅读章节、Trace Ledger、图表、实现参考和自检题。 | 架构、数据流、关键类型、生命周期状态、失败路径和设计权衡。 |
| 可选源码实验 | 使用每章的固定源码链接做验证。 | 已经在书中解释过的事实位于源码中的哪一行。 |

默认路径是第一条。源码链接是证据，不是作业。每章的“继续读源码”部分用于验证细节；除非你在做审计，否则不需要先打开源码才能理解正文。

## 30 分钟路径

需要快速建立心智模型时，按这个顺序读：

1. 先读首页架构图。
2. 读[第 4 章](chapter-04)，掌握协议词汇。
3. 读[第 5 章](chapter-05)，理解会话 facade。
4. 读[第 6 章](chapter-06)，理解 turn loop。
5. 读[第 9 章](chapter-09)和[第 10 章](chapter-10)，理解审批与沙箱。
6. 读[实现参考](implementation-reference)里的总表。

读完后，你应该能解释一个用户请求如何变成 session submission、模型流、一次或多次工具调用、审批或沙箱决策、结构化事件，以及客户端可见输出。

## 2 小时路径

如果你想不打开源码也获得接近源码读者的知识，按这个顺序读：

1. 按顺序读前言和十二章正文。
2. 每章结尾先回答“自检题”，再把源码练习当成可选验证。
3. 反复追问 Trace Ledger：请求现在在哪里，由什么数据结构携带，谁拥有下一步决策，这里可能怎么失败。
4. 最后读[实现参考](implementation-reference)、[模式索引](patterns)和[源码索引](source-atlas)。

这条路径的目标不是记住文件名，而是记住实现的稳定事实。

## 源码审计路径

只有当你想验证本书和源码是否一致时，才需要这条路径：

1. 先读一遍[源码索引](source-atlas)。
2. 只打开每章 Evidence Map 中列出的链接。
3. 用这些文件验证本章的 Trace Ledger。
4. 如果正文无法支持 Trace Ledger，应当把它视为书的问题，而不是读者少读了源码。

## 如何使用自检题

每道自检题都应该能从正文回答。如果必须打开源码才能回答，说明章节解释还不够。后面的源码练习只是提供行级验证。

## 贯穿全书的 Trace Ledger

每个主要子系统都围绕同四个问题解释：

| 问题 | 为什么重要 |
| --- | --- |
| 用户请求现在在哪里？ | 避免停留在抽象架构词。 |
| 什么数据结构携带它？ | 把行为绑定到具体 Rust 类型。 |
| 谁拥有下一步决策？ | 区分入口、协议、运行时、工具和客户端。 |
| 这里可能怎么失败？ | 让拒绝、重试、取消和错误上报变得可见。 |

如果你能在每章回答这四个问题，就已经掌握了一个认真读源码的人会保留下来的核心知识。
